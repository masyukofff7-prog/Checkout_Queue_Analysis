Логика подготовки данных в Power Query

В проекте Power Query использовался для объединения исходных таблиц, приведения типов данных и подготовки данных к дальнейшему анализу очередей.

1. Объединение таблиц

Таблица `store_checkout_queues` была объединена с таблицей `store_stores` по полю `store_uuid`, чтобы получить `store_id` для каждой кассовой операции.

После объединения были отобраны только нужные магазины:
- `98451680`
- `12864064`

Также на этом этапе были приведены типы данных, включая преобразование текстовых полей с датой и временем в формат datetime.

В результате была подготовлена таблица со следующими полями:
- `checks_number`
- `store_id`
- `employees_id`
- `quantity`
- `selling_price`
- `checkout_id`
- `start_operation_dt`
- `end_operation_dt`

Основные функции Power Query, использованные на этом этапе:
- `Table.TransformColumnTypes`
- `DateTime.FromText`
- `Table.NestedJoin`
- `Table.ExpandTableColumn`
- `Table.SelectRows`

2. Подготовка логики определения очередей

Для каждой пары `store_id` и `checkout_id` данные были отсортированы по `start_operation_dt`.

Далее для каждой операции определялось время начала следующей операции на той же кассе. На основе этого рассчитывался разрыв между операциями в секундах:

`gap_to_next_start_s = Duration.TotalSeconds(next_start - current_end)`

Очередь считалась присутствующей, если следующая операция начиналась не позднее чем через 5 секунд после завершения предыдущей:

`gap_to_next_start_s <= 5`

Последовательные операции с таким условием объединялись в единые периоды очереди.

Для каждого периода рассчитывались:
- `queue_start_dt` — начало периода
- `queue_end_dt` — окончание периода
- `checks_with_queue` — количество чеков
- `queue_duration_s` — длительность периода в секундах

Основные функции, использованные на этом этапе:
- `Table.Sort`
- `Table.AddIndexColumn`
- `List.Accumulate`
- `Table.Group`
- `Table.Combine`
- `Duration.TotalSeconds`

Итог

Power Query использовался как инструмент подготовки и преобразования данных перед расчётом периодов очереди. Это позволило объединить источники, привести данные к нужному формату и реализовать логику предварительной обработки в удобном пошаговом виде.

# Анализ очередей на кассах

## Описание задачи

Проведён анализ кассовых операций на основе таблиц `store_checkout_queues` и `store_stores` с целью определения периодов, когда на кассах присутствовала очередь.

---

## Используемые данные

- `store_checkout_queues` — данные по операциям на кассах (чеки)
- `store_stores` — справочник магазинов

---

## Этапы решения

### 1. Объединение данных

Таблицы `store_checkout_queues` и `store_stores` объединены по полю `store_uuid` для получения идентификатора магазина (`store_id`) для каждого чека.

После объединения выполнена фильтрация по магазинам:
- 98451680
- 12864064

Также выполнено приведение типов данных, включая преобразование текстовых дат и времени в формат datetime.

---

### 2. Формирование итоговой таблицы

В результате объединения получена таблица со следующими полями:

- checks_number — номер чека  
- store_id — идентификатор магазина  
- employees_id — идентификатор сотрудника  
- quantity — количество  
- selling_price — цена продажи  
- checkout_id — идентификатор кассы  
- start_operation_dt — время начала операции  
- end_operation_dt — время окончания операции  

---

### 3. Логика определения очереди

Для определения периодов очереди использован прокси-подход.

Очередь считается присутствующей, если следующая операция на той же кассе начинается не позднее чем через 5 секунд после завершения предыдущей:

```
gap_to_next_start_s <= 5
```

Для этого:
- данные сортируются по `store_id`, `checkout_id` и `start_operation_dt`
- с помощью оконной логики определяется время начала следующей операции
- рассчитывается разрыв между операциями
- выделяются интервалы с минимальным разрывом

Последовательные интервалы объединяются в единые периоды очереди.

---

### 4. Расчёт метрик очереди

Для каждого периода очереди рассчитываются:

- `queue_start_dt` — начало периода  
- `queue_end_dt` — конец периода  
- `checks_with_queue` — количество чеков  
- `queue_duration_s` — длительность периода (в секундах)  

---

## Ограничения подхода

В исходных данных отсутствует информация о времени прихода покупателей в очередь.

В связи с этим очередь определяется косвенно — через минимальный разрыв между соседними операциями на одной кассе.

Данный подход позволяет выявить периоды высокой загрузки кассы, но не отражает фактическую длину очереди.
